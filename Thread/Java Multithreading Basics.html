🚦 What is Multithreading?

Multithreading means running multiple tasks (called threads) at the same time in a program.

🧠 Think of your brain handling:

  • Listening to music 🎧

  • Reading a message 📩

  • Breathing 😮‍💨

You do all at once! That’s what multithreading lets your Java program do.

🧵 What is a Thread in Java?

A thread is like a mini-program inside your main program.

In Java, you create a thread by either:

  1. Extending Thread class

  2. Implementing Runnable interface

Example:

java
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running...");
    }
}

🤝 What is Synchronization?

Let’s say 2 people (threads) try to write to a diary (shared resource) at the same time. Pages will get messed up!

Synchronization ensures only one thread at a time can access the diary.

✅ It prevents data conflict and ensures thread safety.

Example:

java
class Counter {
    int count = 0;

    synchronized void increment() {
        count++;
    }
}

Only one thread can run increment() at a time.

🔑 Synchronized Block vs Method:

  • synchronized void method() — locks the whole method

  • synchronized(this) — locks a specific part (better performance)

java
void increment() {
    synchronized(this) {
        count++;
    }
}

📊 What is Thread Priority?

Java lets you set importance for threads:

  • MIN_PRIORITY = 1

  • NORM_PRIORITY = 5 (default)

  • MAX_PRIORITY = 10

Higher priority threads may run before lower ones, but not always. JVM (Java Virtual Machine) decides based on CPU.

Example:

java
Thread t1 = new Thread();
Thread t2 = new Thread();

t1.setPriority(Thread.MAX_PRIORITY);
t2.setPriority(Thread.MIN_PRIORITY);

🧩 Real-Life Analogy:

  • Threads = People standing in line for food 🍽️

  • Synchronization = One person allowed in the kitchen at a time 🧑‍🍳

  • Priority = VIP pass holders might get food first 🎟️

ThreadMini task/program
SynchronizationAvoid resource conflicts
PrioritySet execution importance of thread

🧵 First, What is a Thread Again?

A Thread is like a worker in your program.

Imagine you have a restaurant 👨‍🍳:

  • The chef is the main thread

  • Waiters are other threads (they can take orders while the chef cooks)

Now in Java, there are two main ways to create these "waiter threads".

🏗️ Option 1: Extending the Thread Class

📌 What It Means:

You create your own class that extends Java’s built-in Thread class.
You override the run() method — that’s where your thread’s job goes.

✅ Code Example:

java
class MyThread extends Thread {
    public void run() {
        System.out.println("Thread is running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread t1 = new MyThread(); // thread created
        t1.start(); // thread started
    }
}

🧠 In Simple Terms:

You’re saying:
"Hey Java, I’ll make my own version of a thread. Here’s what it should do when it runs!"

🧩 Option 2: Implementing Runnable Interface

📌 What It Means:

You don’t extend Thread, instead you create a class that implements Runnable, and pass it to a Thread object.

✅ Code Example:

java
class MyRunnable implements Runnable {
    public void run() {
        System.out.println("Runnable thread running...");
    }
}

public class Main {
    public static void main(String[] args) {
        MyRunnable myJob = new MyRunnable();
        Thread t1 = new Thread(myJob); // pass job to thread
        t1.start(); // start the thread
    }
}

🧠 In Simple Terms:

You're saying:
"I’ll just tell Java what work needs to be done (via Runnable), and then I’ll give that job to a Thread to handle it." 💼🧵

🆚 Thread Class vs Runnable Interface – Simple Comparison

FeatureThread ClassRunnable Interface
Inheritance used?Yes (extends Thread)No (implements Runnable)
Can extend another class?❌ No (Java only allows one superclass)✅ Yes (you can extend other classes too)
FlexibilityLess flexibleMore flexible
How to start the thread?t1.start()new Thread(obj).start()

🍽️ Real-Life Analogy

Imagine a restaurant:

  • Thread class = You hire a waiter and give them a task directly.

  • Runnable interface = You write a "task note", and hand it to any available waiter.

Using Runnable is like writing instructions (task) and passing it to someone to run — more flexible!

🎯 When to Use What?

  • Use Thread class when:

    • You don’t need to extend another class

    • Quick, simple tasks

  • Use Runnable interface when:

    • You want more flexibility

    • You already extend another class

    • You're working on a bigger app or using thread pools

BUT the key difference is how responsibilities are separated and how Java handles them under the hood.

⚙️ Think of it Like This:

✅ Thread = Worker + Job

When you extend the Thread class, you are saying:
🧑‍🔧 “I am the worker AND I know what to do!”

java
class MyThread extends Thread {
    public void run() {
        System.out.println("I'm doing the job!");
    }
}

But that means:
❌ You can’t hire the same worker (Thread) to do another task
❌ You can’t inherit another class (Java only supports single inheritance)

✅ Runnable = Job Only

When you implement Runnable, you’re saying:
🧾 “Here’s the job – now any Thread can do it!”

java
class MyJob implements Runnable {
    public void run() {
        System.out.println("This is my job.");
    }
}
Thread t = new Thread(new MyJob());
t.start();

You’re separating:

  • The task (Runnable)

  • The executor (Thread)

This gives you:
✅ More flexibility
✅ Cleaner code
✅ Better use in big projects, thread pools, and executors (used in real-world apps)

🧪 Behind the Scenes

  • Thread class implements Runnable internally.

  • So even if you extend Thread, it still uses the Runnable interface under the hood.

java
public class Thread implements Runnable {
   // Java source code
}

🧠 So, when you pass a Runnable to a Thread, Java is doing the same thing internally!

🧩 Analogy: Worker and Job Card

Thread classA person with one fixed job
RunnableA job card given to any worker

Imagine:

  • 🧍 Thread = Worker

  • 📝 Runnable = Task sheet

With Runnable, you can give different task sheets to different workers.
With Thread, the worker has one fixed job built into them.

🔄 When to Use Which?

Use CaseUse ThreadUse Runnable
Quick & simple job✅✅
Need to extend another class❌✅
Want separation of task & thread❌✅
Using thread pools or Executors❌✅

🧠 Final Thought

Java keeps both because:

  • Thread is simple for beginners and small tasks

  • Runnable is better for professional, scalable apps

They do the same thing, but Runnable gives more control and is preferred in real-world applications.

🧵 Java Threads – Summary Notes

🔹 What is a Thread?

A thread is a small, independent unit of execution in a program. Java allows multithreading – running multiple threads at the same time.

🧱 Two Ways to Create a Thread:

1. Extending the Thread Class

java
class MyThread extends Thread {
    public void run() {
        // task to perform
    }
}
MyThread t = new MyThread();
t.start();

  • Inherits Thread class

  • Override run() method

  • Start with start() method

  • ❌ Can’t extend any other class

2. Implementing Runnable Interface

java
class MyTask implements Runnable {
    public void run() {
        // task to perform
    }
}
Thread t = new Thread(new MyTask());
t.start();

  • Implements Runnable interface

  • Pass object to Thread constructor

  • ✅ Can extend other classes

  • ✅ Preferred for flexibility

🔍 Key Differences:

FeatureThread ClassRunnable Interface
InheritanceExtends ThreadImplements Runnable
Code ReusabilityLess flexibleMore flexible
Supports Multiple Inheritance❌ No✅ Yes
Used in Thread Pools❌ Not preferred✅ Preferred

🧠 Why Use Runnable?

  • Clean separation between task and thread

  • More flexible in large apps

  • Works well with Executors, thread pools, etc.

💡 Extra Note:

Even the Thread class itself implements Runnable.
